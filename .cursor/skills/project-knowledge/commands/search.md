# search — 关键词检索

## 触发条件
- 用户输入 `/project-knowledge search 关键词` 或 `/project-knowledge 检索 关键词`

## 执行步骤

---

### 第一步：直接搜索索引

**1.1 搜索 Index.md**
- `Grep` 在 `docs/current/Index.md` 中搜索关键词，`-i` 忽略大小写，输出模式 `content`
- 获取匹配行及其上下文（模块名、文档标题、摘要）

**1.2 索引命中处理**
若找到匹配：
- 提取匹配文档的：文件路径、类型、状态、摘要
- 评分规则（按优先级从高到低）：
  - 标题完全匹配 ★★★★★
  - H2/H3 标题匹配 ★★★★
  - 正文/摘要匹配 ★★★
- **直接进入第四步输出**

---

### 第二步：转换关键字再搜索索引

**2.1 关键字转换**
根据关键词类型进行同义扩展：
- 英文 → 中文（如 `OAuth2` → `认证`、`JWT` → `令牌`）
- 中文 → 英文（如 `认证` → `OAuth2/Authentication`，`升级` → `Upgrade`）
- 别名/缩写 → 全称（如 `SSO` → `单点登录/Single Sign-On`）
- 功能描述 → 模块名（如 `浏览器` → `Browser`，`收藏夹` → `Bookmark`）

**2.2 再次搜索 Index.md**
- 用转换后的关键字重新执行 `Grep` 搜索 `docs/current/Index.md`
- 若找到匹配，按 1.2 规则评分处理
- **命中则进入第四步输出**

---

### 第三步：阅读索引理解匹配

**3.1 读取 Index.md 全文**
- `Read` 完整读取 `docs/current/Index.md`

**3.2 语义匹配**
理解用户搜索意图，在索引中寻找语义相关的文档：
- 模块名相关：`Electron`、`IronClaw`、`OnlineHub` 等
- 功能描述相关：如"启动"匹配 Startup 模块、"布局"匹配 PageLayout 模块
- 状态筛选：`已完成`、`待审核`、`已归档` 等

**3.3 评分输出**
- 精确模块匹配 ★★★★
- 功能相关匹配 ★★★
- **命中则进入第四步输出**

---

### 第四步：全文搜索兜底（最终兜底）

**4.1 全文搜索**
- `Grep` 在 `docs/current/` 中搜索关键词，`-i` 忽略大小写，输出模式 `files_with_matches`
- 排除 `Index.md`、`SPEC_Engineering.md`、中文搜索时排除 `SPEC_.*`

**4.2 收集详情**
- `Read` 每个匹配文件前 30 行提取标题、类型、状态
- `Grep -C 2` 获取关键词上下文

**4.3 评分**
- 标题完全匹配 ★★★★★
- 文件名匹配 ★★★★
- H2/H3 标题匹配 ★★★★
- 正文匹配 ★★★
- 代码块匹配 ★★

---

### 第五步：输出结果

若无匹配：
```
## 检索结果: "{keyword}"

未找到匹配文档。

建议：
- 尝试其他关键词
- 运行 /project-knowledge build 查看所有模块
```

若有匹配，输出：
```
## 检索结果: "{keyword}"

找到 N 个匹配文档（来源：索引/语义匹配/全文）：

### 1. ★★★★★ 认证前端设计
|- [文件](common/Authentication/DESIGN_FE_Authentication.md) | 类型: DESIGN_FE | 状态: 已完成
|- 摘要: OAuth2 回调处理、登录状态管理、登出流程。
|- 匹配位置: 标题中包含 "{keyword}"
```

按 ★ 等级降序排列，同级按模块字母序。最多显示 15 个结果。
