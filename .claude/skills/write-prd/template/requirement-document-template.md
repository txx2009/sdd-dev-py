「格式示例」仅示范形态，勿照抄业务字句。

## 一、需求文档正文结构（章节骨架）

以下 **`###` 标题** 固定，不得增删改序。

> **编写要求（总则）**
>
> - **骨架**：与各 `###` 完全一致；示例嵌于各节。
> - **可验收**：交互与数据规则可验收；不要求颜色字体间距。
> - **禁止实现细节**：禁止正文写组件库名、标签、属性/事件/API（如 `el-*`、`v-model`、`type="primary"`）；用**数据表格**、**主按钮**、**抽屉**、**分页器**等；例外仅附录/契约等非正文（若制度允许）。
> - **通俗易懂**：短句、术语与 **`### 枚举`** 及字段表一致；概述与工具栏用「用户动作 + 系统反馈」；代号首次释义；分支用「若…则…」；「说明」列可照着测；**不删减**表格、条件、错误提示与权限边界。
> - **字段表（列表 / 查询 / 表单共用）**：表格化；须含 **默认值**、**是否唯一值**；列至少含字段名、类型、必填、默认值、是否唯一值、数据来源、说明（或校验）；无语义填 `—`；可增长度限制。**字段名称**列仅写**中文业务名**，不尾随英文 key（如 `` `reportNo` ``）；若需与研发对齐，在**说明**列写「对应数据键：…」或放附录。形态校验的完整场景、拦截与提示以 **`### 数据验证规则`** 为准。

### 功能概述

> **编写要求**：目标与用途一笔写清；与兄弟页分文档时一句分工；避免纯口号。

**示例**：`本模块用于…（勿抄占位）`

### 角色权限

> **编写要求**：**必有**；数据权限（单位/区域/行业/全局）+ 角色功能权限（增删改查）；无差异写「不适用」及理由。

**示例**：维度表 + 角色 × 增删改查矩阵（参照已评审范例）。

### 页面结构

> **编写要求**
>
> - **必有线图**（框线 / Mermaid / 与树配套），覆盖主区、查询/工具栏/主内容/分页、主要交互；禁止纯散文或仅「左中右」；弹层标打开方式。
> - **含左树**：须同时满足本节与 **`### 目录树`**（结构图 + 节点模型 + 联动/加载等）；禁止仅「支持树展示」。

```text
┌────────────────────────────────────────┐
│ 标题 / 面包屑                           │
├────────────────────────────────────────┤
│ 查询区  [条件] [查询] [重置]           │
├────────────────────────────────────────┤
│ 工具栏  [主操作] …                     │
├────────────────────────────────────────┤
│ 主内容区（列表 / 表单 / 分栏）          │
├────────────────────────────────────────┤
│ 分页                                   │
└────────────────────────────────────────┘
```

### 枚举

> **编写要求**：静态码表全集，`#### 枚举：<含义>` + 表；**表头与取值列名用中文**（首列推荐「存储值」对应持久化/接口取值，次列「展示名」）；无则「不适用」并说明原因。

**示例**：`#### 枚举：状态` — 三行「存储值 / 展示名 / 说明」表。

### 目录树

> **编写要求（有树页 · 条件强制）**：须在「页面结构」体现布局，并给出 **层级结构图** + **树节点字段表** + 根/层级/联动/懒加载/树上操作/初态；无子项写不适用。

| 维度 | 要求 |
|------|------|
| 树形层级结构图（强制） | 在「页面结构」内或紧接树数据子节；缩进树或 Mermaid；禁止仅用字段表代替；节点文案与字段表一致。 |
| 节点数据模型 | 节点标识、父标识、显示名、排序、类型/是否叶子等；列含类型、必填、默认值、唯一性/校验。 |
| 层级与根 | 最大深度、多根、虚拟根、同级排序、父子表达。 |
| 与主区联动 | 选中/展开过滤列表或详情；未选中行为；选中态是否持久。 |
| 加载 | 全量或懒加载；懒加载须触发条件与来源。 |
| 树上操作 | 增删改拖拽等；与角色权限一致；删父对子规则。 |
| 初态 | 默认展开/选中、空树占位。 |

### 查询功能

> **编写要求**：查询字段表（列同总则「字段表」）；写清查询/重置/回车等行为。

### 列表展示

> **编写要求**：**至少一张**列表字段表，或「不适用」+ 理由；禁止仅「支持分页排序」；多专题分表。列至少：字段名、类型、是否必填、默认值、是否唯一值、数据来源、说明（或校验）。无列表写不适用。可增「长度限制」列。

### 列表卡片

> **编写要求（条件强制）**：有卡片/列表切换则**必有**：框线图 + 结构 + 交互 + 与列表关系。

### 工具栏按钮

> **编写要求**：名称、主次、显隐、打开方式、结果；「用户动作 + 系统反馈」。

### 表单设计

> **编写要求**：字段与校验；字段表列同总则；抽屉/侧滑写入口、分区、主操作/取消、未保存关闭确认；联合唯一写在说明；无不适用须声明。

### 流程图

> **编写要求**：须写清可执行规则与失败提示；**须同时使用** Mermaid 流程图与**编号步骤列表**（列表与 Mermaid 语义一致，便于评审与测试对齐）；无不适用须声明。

### 导入导出

> **编写要求（含导入/导出时 · 条件强制）**
>
> - **仅 Excel**（`.xlsx`/`.xls`）；先确认字段再落盘；须含：导入说明（模板列、失败行/部分成功）、**导入字段表**、导出说明、**导出字段表**；列至少：名称、key、类型、必填（导入）、默认值、是否唯一值、数据来源、校验/错误提示（导入）、脱敏（导出）；错误提示与数据验证**同一列不得两套文案**。
> - **若无**导入/导出：本节「不适用」及理由。

### 数据验证规则

> **编写要求**
>
> - 正则**按字段**分 **`#####`** 小节。
> - 子结构须落在本 `###` 下，不新增同级 `###`：
>   1. **`#### 校验范围与场景`**
>   2. **`#### 正则形态校验（按字段）`** — 每字段 **`##### <中文名>（`key`）`**：适用场景、前置（若有）、**单独一个** fenced 代码块（语言标记 `regex`）、匹配说明、错误提示
>   3. **`#### 其它验证规则（非正则）`** — 编号或分段，**不用大表**
>   4. **`#### 跨字段与业务规则`**（可选）
>   5. **`#### 规则汇总（验收清单）`**
> - **无正则字段**：在 **`#### 正则形态校验（按字段）`** 写「**本页无正则校验字段**」及理由。

**结构示例**：

#### 正则形态校验（按字段）

##### 统一社会信用代码（`uscc`）

1. **适用场景**：新增、编辑、Excel 导入。
2. **正则表达式**：

```regex
^[0-9A-HJ-NPQRTUWXY]{2}\d{6}[0-9A-HJ-NPQRTUWXY]{10}$
```

3. **错误提示**：统一社会信用代码格式不正确。

（无正则时：在本 `####` 下写「本页无正则校验字段」并转入 **`#### 其它验证规则（非正则）`**。）

### 注意事项

> **编写要求**：仅业务/交互/验收边界；不含文档路径/README/skill 名、文档管理条文、UI 像素与组件库样式。
